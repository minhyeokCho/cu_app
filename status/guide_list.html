📢 [CSS 규칙]

👉 괄호 열기, 닫기 앞 뒤로 띄어쓰지 않음
👉 콜론 띄어쓰지 않음
👉 속성이 끝난 후 세미콜론은 붙여 씀, 다음 속성과는 띄어 씀

예시) .bg_box_flx {display:flex; overflow:hidden; flex-wrap:wrap; margin-top:-11px;}

👉 margin, padding 두개 부터 축약 사용

잘못된 예) margin:0 10px 0 0; (X) 올바른 예) margin-right:10px; (O) padding-top:10px padding-bottom:10px (X) padding:10px 0 (O)

👉 기존에 선언한 margin, padding 값을 다시 선언할 경우엔 축약을 사용하지 않는다.

- 기존 padding:0 24px 로 선언되어 있는 속성을 padding:10px 24px 10px 24px 로 수정해야 한다면 - padding-top:10px; padding-bottom:10px; 로 선언 하기.

👉 transition 속성은 밑 줄에 따로 작성

👉 1뎁스 메뉴별로 CSS 파일 생성, html에 추가. 한 메뉴 동시 작업할 때는 CSS 분리 후 합치기.

👉 팝업 CSS는 하단에 작성.

📢 [클래스 규칙]

👉 클래스는 소문자, 언더바 ( _ ) 사용. 카멜표기 사용 안함.

👉 레이아웃클래스 (모양을 잡기 위해 선언한 클래스)를 먼저 선언한 후 의미에 맞는 클래스를 선언하거나 감싸주기.

<div class="icon_sprite_32 cart btn_menu"></div> ㄴ icon_sprite_32 cart : 아이콘 모양 전용 class ㄴ btn_menu : 메뉴 버튼 관련 class <div class="add_info"> <!-- 주소정보 --> <div class="box_shape bg_gray01"> <div class="cotn"> </div> </div> <!--// 주소정보 --> </div> ㄴ add_info : 정보 제공용 class ㄴ box_shape bg_gray01 : 박스영역 스타일 class

👉 레이아웃 클래스로 의미 있는 영역의 스타일을 다시 잡는건 지양할 것.

📢 [헤더 사용방법]

👉 메뉴별 클래스에 상속 시켜 노출되어야 하는 아이콘을 display block 추가 (요소 기본 스타일 - dispaly:none)
